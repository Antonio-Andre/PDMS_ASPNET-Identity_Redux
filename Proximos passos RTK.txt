1. Criar a Navbar e Rotas

Instalar o react-router-dom.

Criar uma Navbar fixa no topo para navegar entre:

/dashboard (Resumo com os novos Cards).

/companies (A Tabela de Gestão que planeámos).

/vans e /employees.

2. Transformar a Lista em Tabela Editável
Como já tens o objeto auth no MockDb, vamos implementar a lógica de permissões real:

Modo Leitura: Todos os utilizadores veem os dados.

Modo Edição: Se currentUser.role === 'Admin', a tabela mostra botões de "Editar" e "Eliminar".

Mapeamento: Corrigir definitivamente os campos para que taxId, email e location apareçam sem os traços vazios.

Carregue o utilizador do MockDb no arranque.

Permita "trocar de utilizador" (ex: mudar de Admin para Driver) para testares se a interface bloqueia ou permite ações conforme a Role.

Para a página inicial (Dashboard), em vez de uma tabela densa, vamos criar Cards visuais para as 5 empresas. Cada Card mostrará o nome, localização e um indicador de "Status" ou "Frota Ativa".
